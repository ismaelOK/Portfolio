Sujet et démarche

La veille technologique est une démarche continue : surveiller les évolutions d'un domaine, identifier les technologies montantes et adapter ses pratiques. Pour une entreprise dans le secteur du logiciel, c'est une condition pour rester compétitif et éviter la dette technique.

J'ai choisi de me concentrer sur les APIs graphiques — OpenGL, Vulkan, WebGL, WebGPU — parce que c'est à l'intersection de ce qui m'intéresse : la programmation bas niveau et la création visuelle. Le domaine est en pleine mutation depuis quelques années, ce qui en fait un sujet de veille actif et riche.

Sources

  • Serveurs Discord spécialisés — la principale source au quotidien. Des communautés d'enthousiastes et de professionnels (graphistes, développeurs de moteurs, demosceners) qui partagent leurs expériences, discutent des dernières sorties (nouvelles extensions Vulkan, avancement de WebGPU dans les navigateurs, benchmarks GPU) et publient leurs créations. C'est là que l'actualité arrive en premier, avant les articles.
  • Shadertoy (shadertoy.com) — plateforme de shaders GLSL en temps réel. Permet d'expérimenter des techniques (ray marching, SDF, fluid simulation) directement dans le navigateur et de voir ce que d'autres font.
  • learnopengl.com & vulkan-tutorial.com — références pour comprendre les APIs en profondeur, pas juste les utiliser en surface.
  • YouTube : TheCherno (série moteur de rendu en C++), Brendan Galea (Vulkan pas à pas), Sebastian Lague (experiments visuels).
  • Documentation officielle : Khronos Group (OpenGL, Vulkan), W3C (spec WebGPU), MDN (WebGL).

Panorama des APIs

Une API graphique est l'interface entre le code et le GPU — elle définit comment envoyer géométrie, textures et instructions de rendu au matériel.

1992 · Toujours présent

OpenGL

Standard historique multiplateforme. Plus maintenu activement depuis 2017 mais encore très répandu dans les outils CAO, éducatifs et embarqués. Point d'entrée naturel pour apprendre le rendu 3D.

MatureC/C++
2016 · Standard montant

Vulkan

Successeur d'OpenGL, bas niveau : gestion explicite de la mémoire, command buffers, synchronisation. Verbeux mais très performant. Adopté par les jeux AAA et les moteurs (Unreal, Unity).

Haute perf.C/C++
2011 · Standard web

WebGL / WebGL 2

OpenGL ES dans le navigateur via JavaScript. Utilisé par Three.js, Babylon.js, et les expériences artistiques interactives. La démo ci-dessus tourne en WebGL.

NavigateurJS / GLSL
2023 · Futur du web

WebGPU

Nouvelle API W3C inspirée de Vulkan/Metal/DX12, disponible dans Chrome depuis 2023. Compute shaders, WGSL, performances proches du natif. Three.js et Babylon.js migrent progressivement.

ÉmergentJS / WGSL
Windows / Xbox

Direct3D 12

API Microsoft bas niveau, équivalent de Vulkan sur l'écosystème Windows. DXR (DirectX Raytracing) a popularisé le ray tracing temps réel depuis 2018.

Metal

API Apple, remplace OpenGL sur iOS/macOS depuis 2018. Très intégrée aux GPU M1/M2/M3. Utilisée par Blender, Affinity et les portages via MoltenVK.

Les serveurs Discord restent ma source principale — c'est là que les discussions techniques arrivent en temps réel, avant les articles. Shadertoy m'a permis de passer de la lecture à la pratique : expérimenter des techniques comme le ray marching (illustré dans la démo ci-dessus) sans infrastructure complexe.

Ce que je retiens surtout, c'est la distinction entre les niveaux d'abstraction : WebGL pour prototyper rapidement dans le navigateur, OpenGL pour apprendre les fondamentaux, Vulkan quand la performance prime. Savoir à quel niveau se placer selon le contexte, c'est en soi une compétence.
