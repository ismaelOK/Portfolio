M2L — Gérer le patrimoine informatique

Le projet M2L a mobilisé plusieurs sous-compétences de gestion du patrimoine informatique.

Exploiter des référentiels, normes et standards
  • Développement en PHP selon le modèle MVC fourni par le lycée
  • Respect des conventions de nommage SQL et des contraintes de la base de données existante
  • Utilisation des normes HTML/CSS et des bonnes pratiques JavaScript
Mettre en place et vérifier les niveaux d'habilitation
  • Mise en place de rôles distincts : utilisateur standard / responsable de formation
  • Vérification des droits d'accès à chaque action (accepter/refuser une demande d'inscription)
  • Protection des routes sensibles selon le rôle connecté
Gérer des sauvegardes
  • Versionnage du code source sur GitHub avec commits réguliers et branches par fonctionnalité
  • Sauvegarde de la base de données MySQL lors des livraisons

API ACC — Gérer le patrimoine informatique

Le développement de l'API e-sport a mobilisé des compétences de gestion du patrimoine applicatif.

Exploiter des référentiels, normes et standards
  • Respect des spécifications d'endpoints définies en amont (structure, méthodes HTTP, codes de retour)
  • Application des standards REST pour la conception de l'API
  • Utilisation du standard JWT (RFC 7519) pour l'authentification — token Bearer, HMAC-SHA256, expiry 1h
  • Hash des mots de passe avec Argon2id (PASSWORD_ARGON2ID) conformément aux recommandations de sécurité actuelles
Gérer des sauvegardes
  • Versionnage continu du code sur GitHub avec branches dédiées par fonctionnalité
  • Sauvegarde de la base de données MySQL lors des jalons de développement
